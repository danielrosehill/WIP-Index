# Scope — WIP Index

## What belongs here

Repositories that contain **actual work in progress** — projects being actively built, partially implemented, or paused mid-development with a clear intent to continue.

Include a repo if it is:

- Under active development with committed code
- Partially built and awaiting further work (paused, not abandoned)
- A usable-but-incomplete tool that still needs features, polish, or bug fixes
- An experimental build where the code itself is the artifact being iterated on

The defining test: **there is code (or equivalent build artifacts), and it is not finished.**

## What does NOT belong here

- **Plan-only / spec-only repos with no implementation** → use `Architectural-Plans-Index`
- **Reference documentation, notes, tutorials** → use `Docs-And-Notes-Index`
- **Completed and shipped projects** → use the relevant topical index
- **Abandoned projects with no intent to resume** → do not index, or use an archive index
- **Pure ideation repos with no code** → use `Architectural-Plans-Index`

## Overlap rules

- A WIP repo can also live in a topical index (e.g. `MCP-Projects-Index`) reflecting its subject matter
- A WIP repo can appear in `Architectural-Plans-Index` as well if it contains a substantial planning doc alongside the implementation
- When a WIP project ships, remove it from this index and leave it in its topical index(es)
